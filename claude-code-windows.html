Claude Code 版マニュアル（Windows）

Claude Codeに
外部記憶を導入する

導入プロンプトをClaude Codeに貼るだけで、Obsidianのインストールから設定まで、Claude Code自身がセットアップしてくれます。

所要時間 約20分 🪟 このページは Windows 用
🍎

Macをお使いの方へ：このページはWindows用です。Mac専用の手順ページはこちら。

Mac版マニュアルへ →
01

事前準備

  • Windows 10 / 11
  • Claude Codeが使えること（デスクトップアプリのCodeモード、またはターミナルのCLI版どちらでもOK）

ObsidianのインストールはClaude Codeが自分でやってくれるので、事前に入れておく必要はありません。すでに入っていても問題ありません。

02

導入 — プロンプトをコピペするだけ

下の「コピーする」を押して、そのままClaude Codeに貼り付けて送信してください。あとはClaude Codeが順番に作業してくれます。途中で実行の許可を求められたら、内容を見て許可してあげてください。

installer-claude-code-windows.md 
あなたに「外部記憶システム」を導入してもらいます。
Obsidian（ローカルのメモアプリ）を外部記憶として使い、セッションをまたいで記憶を引き継げるようにする仕組みです。

環境はWindowsです。以下の手順を上から順番に実行してください。コマンドを実行する前に何をするか一言説明し、エラーが出たら止まって私に報告してください。

## Step 1: Obsidianの確認とインストール

1. Obsidianがインストール済みか確認する
2. なければ winget install -e --id Obsidian.Obsidian でインストールする
3. wingetが使えなければ、その旨を私に伝えて https://obsidian.md からの手動ダウンロードを案内する

## Step 2: 記憶用フォルダ（Vault）の作成

1. まず実際の「ドキュメント」フォルダの場所を確認する。
   OneDriveでドキュメントフォルダが移動されている場合があるため、PowerShellで以下を実行して正確なパスを取得すること：

   [Environment]::GetFolderPath('MyDocuments')

2. 取得したドキュメントフォルダの直下に、以下のフォルダを作成する：

Claude-Brain/
├── daily/      （日次の作業ログ）
├── todos/      （todoリスト）
├── ideas/      （アイデア・構想）
├── projects/   （プロジェクトの現在地）
├── decisions/  （判断・ルールの記録）
├── mistakes/   （ミス記録）
└── sessions/   （セッション記録）

3. Claude-Brain/README.md を作成し、以下の内容を書き込む：

- このVaultはAIの外部記憶として機能すること
- 各フォルダの役割（上記の通り）
- 更新ルール：
  - セッション開始時：README + daily最新3件 + projectsを読む
  - 記録タイミング：タスク完了・情報更新・テーマ切替・節目ごとに許可不要で即記録
  - daily/への書き込みは追記（上書きしない）
  - ミスが発生したらmistakes/に即座に記録
  - 重要な判断をしたらdecisions/に記録

## Step 3: MCPサーバーの接続

1. 以下のコマンドを実行する。「ドキュメントパス」はStep 2で取得した実際のパスに置き換えること：

claude mcp add --scope user obsidian-brain -- cmd /c npx -y @modelcontextprotocol/server-filesystem "ドキュメントパス\Claude-Brain"

※ Windowsではnpxを直接指定するとMCPサーバーの起動に失敗することがあるため、必ず cmd /c を挟むこと。

2. claude mcp list を実行して obsidian-brain: ✓ Connected が出ることを確認する

## Step 4: CLAUDE.mdにルールを追記

%USERPROFILE%\.claude\CLAUDE.md （なければ作成）に、以下のセクションを追記する。「ドキュメントパス」は実際のパスに置き換えること：

## 外部記憶（Obsidian）ルール ⭐ 最重要

### Vault場所
ドキュメントパス\Claude-Brain\

### セッション開始時（必須）
新しいセッションが始まったら、必ず最初に以下を読む：
1. Claude-Brain\README.md
2. Claude-Brain\daily\ の最新3件
3. Claude-Brain\projects\ の関連ファイル

### 自動記録ルール（許可不要・常時）
以下のタイミングで、ユーザーに確認せず即座にClaude-Brainに書き込む：

| タイミング | 書き込み先 |
|---|---|
| 新しい情報・決定・方針が出た | daily/YYYY-MM-DD.md に追記 |
| タスクが完了した | daily/YYYY-MM-DD.md に追記 |
| 会話の節目（テーマが変わる時） | daily/YYYY-MM-DD.md に追記 |
| ミスを指摘された | mistakes/YYYY-MM-DD-[内容].md |
| 重要な判断をした | decisions/YYYY-MM-DD-[内容].md |
| セッション終了時 | daily/YYYY-MM-DD.md に総まとめ |

- 許可を取らない。書いたら返答の末尾に（📝 記録済み）と添えるだけでOK
- daily/への書き込みは上書きでなく追記
- 書きすぎを恐れない。抜け漏れより過剰記録の方がよい

## Step 5: 完了報告

すべて終わったら、以下を私に報告してください：

1. 実行した内容のまとめ（Claude-Brainの実際のパスも明記）
2. 私が手動でやること：
   - Obsidianを起動 →「保管庫としてフォルダを開く」→ ドキュメント内の Claude-Brain を選択
   - Claude Codeを再起動（MCP反映のため）
3. 再起動後のテスト方法：新しいセッションで「Claude-Brainを確認して」と送る
⚠️Claude Codeがコマンド実行の許可を求めてきたら、内容を確認してから許可してください。このプロンプトで触るのは「Obsidianのインストール」「Claude-Brainフォルダ」「Claude Codeの設定ファイル」の3つだけです。それ以外を触ろうとしたら止めてOK。
03

動作確認 — 3ステップで完了

1
Obsidianを起動してVaultを開く「保管庫としてフォルダを開く」→ ドキュメント → Claude-Brain を選択。フォルダ一覧（daily, projects…）が左側に見えたらOK。
2
Claude Codeを再起動する一度終了して開き直す。MCP接続はここで反映されます。
3
新しいセッションでテスト下のメッセージを送って、Claude CodeがVaultの中身を読んで報告してきたら成功！
Claude-Brainを確認して
✅「README.mdを確認しました。dailyフォルダはまだ空ですね」のように中身を読んで答えたら接続成功です。「そのようなファイルは見つかりません」と返ってきたらFAQへ。
04

使い方 — 育てるほど賢くなる

🌱 基本の使い方

  • セッション開始時：「昨日の続きやろう」「今の状況教えて」→ Claude Codeが勝手に記憶を読んで文脈を把握
  • 作業中：節目ごとに勝手に記録（返答の末尾に「📝 記録済み」と出ます）
  • 覚えてほしいことがあったら：「これ記録しといて」と言えばOK

🧠 賢く育てるコツ

  • ミスは必ず指摘する → mistakes/に記録され、同じミスを繰り返さなくなる（一番効果が大きい）
  • 「それ違う、こうして」を遠慮しない → 好みやルールがdecisions/に蓄積していく
  • Obsidianのグラフビューを見てみる → 記憶の繋がりが可視化されて楽しい（モチベ維持に効く）
  • 最初から完璧を目指さない → 使いながら「これも記録して」と足していくのが正解
💡応用：Codexも使っているなら、Codex版マニュアル（Windows）の手順も実行すると2つのAIが同じ記憶を共有します。ChatGPTやGeminiなどブラウザ型AIはローカルファイルに直接アクセスできないため対象外です。
05

トラブルシューティング

「Claude-Brainを確認して」と送っても読んでくれない

① Claude Codeを再起動したか確認（MCPは再起動後に反映されます）。
② ターミナルで claude mcp list を実行して obsidian-brain: ✓ Connected が出るか確認。出ない場合は導入プロンプトのStep 3をもう一度実行してもらってください。
③ Windowsでは cmd /c を挟まないとMCPサーバーが起動しないことがあります。「claude mcp listの結果を確認して、obsidian-brainの設定にcmd /cが入っているか調べて」とClaude Codeに聞いてみてください。

Claude-Brainフォルダが見つからない・変な場所にできた

OneDriveを使っていると「ドキュメント」フォルダが C:\Users\名前\OneDrive\ドキュメント に移動されていることがあります。「PowerShellで [Environment]::GetFolderPath('MyDocuments') を実行して、Claude-Brainがそこにあるか確認して」とClaude Codeに頼んでください。MCPの設定パスと実際の場所がズレていると接続できません。

wingetが使えないと言われた

obsidian.md から手動でダウンロードしてインストールすればOKです。それ以外の手順は変わりません。「Obsidianは手動で入れたから、Step 2から続けて」と伝えてください。

npxが見つからないと言われた

Node.jsが入っていない可能性があります。nodejs.org からLTS版をインストールしてから、Step 3をやり直してください。「Node.jsを入れたから、Step 3からやり直して」と伝えればOKです。

自動で記録してくれない

ルールを読み込めていない可能性があります。「CLAUDE.mdの外部記憶ルールを確認して」と送ってみてください。それでも記録しない場合は「今の内容をClaude-Brainのdailyに記録して」と都度言えば確実に記録されます。使ううちに自動化の精度は上がっていきます。

記憶がごちゃごちゃしてきた

順調に育っている証拠です。「Claude-Brainの中身を整理して。古いdailyの要点をprojectsにまとめて」と頼めば、AI自身に大掃除させることができます。月1回くらいがおすすめ。
